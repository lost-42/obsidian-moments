Obsidian 插件需求文档：obsidian-moments
==================================================

1. 项目概述
- 插件名称：obsidian-moments
- 核心目标：开发一款兼容移动端和 PC 端的 Obsidian 插件，实现类似“微信朋友圈”的碎片化记录、照片瀑布流（九宫格）展示以及评论互动功能。
- 技术栈：TypeScript, Obsidian Plugin API, HTML/CSS (需着重适配移动端响应式布局和虚拟键盘交互)。

2. 插件设置界面 (Plugin Settings)
插件需注册一个设置面板，包含以下 3 个配置项：
- Moments 存储路径：文本输入框。存放每一条朋友圈 .md 文件的目标文件夹（例如：Moments/记录/）。如果目录不存在，插件需在写入时自动创建。
- 图片附件存储路径：文本输入框。通过此插件上传的图片存放路径（例如：Moments/Attachments/）。
- 展示顺序：下拉选择框（正序/倒序）。朋友圈瀑布流视图的默认排序方式（按创建时间）。

3. 核心功能一：发布面板 (Create Modal)
注册一个全局命令面板指令：“创建 Moments”。触发后打开一个弹窗面板。

3.1 面板 UI 要求 (需针对移动端优化)
- 标题：单行输入框。
- 地点：单行输入框。
- 内容：多行文本框 (textarea)，需支持高度自适应和内部滚动，防止移动端键盘遮挡。
- 图片选择器：原生输入框，支持多选图片。
- 发布按钮：点击后执行数据保存逻辑，并给出保存成功的提示 (Notice)。

3.2 数据处理与保存逻辑
当用户点击“发布”时，插件需自主接管文件生成：
- 图片处理与保存：
  1. 遍历用户选择的图片文件，读取为 ArrayBuffer。
  2. 生成安全且唯一的文件名：IMG-YYYYMMDD-HHmmss-随机数.后缀。
  3. 使用 Obsidian API (app.vault.createBinary) 将图片物理保存到设置的附件路径下。
- 生成 Markdown 记录文件：
  1. 文件名规则：Moments-YYYY-MM-DD-HHmmss-标题.md。
  2. 使用 app.vault.create() 在 Moments 存储路径下创建文件。
  3. 文件内容结构要求（YAML 元数据严格使用中文）：
     ---
     标题: 用户输入的标题
     地点: 用户输入的地点
     创建时间: YYYY-MM-DD HH:mm:ss
     更新时间: YYYY-MM-DD HH:mm:ss
     评论: []
     ---
     (这里是用户输入的正文内容)

     ![[IMG-20260228-120001.png]]

4. 核心功能二：动态视图渲染 (Feed View)
注册一个 Markdown 代码块处理器 (Code Block Processor) 来动态生成朋友圈视图，避免向原始 md 文件写入臃肿的 HTML。

- 触发语法：代码块标记为 moments
- 数据解析逻辑：
  1. 插件读取 Moments 存储路径下的所有 .md 文件。
  2. 提取每个文件的中文元数据（标题、地点、创建时间、评论）和正文内容（提取文本和图片链接）。
  3. 根据设置中的展示顺序，按“创建时间”进行排序。
- UI 渲染逻辑 (仿微信朋友圈)：
  1. 构建 DOM 树，每条记录渲染为一个 Card。
  2. 头部：展示默认头像、标题、地点标签、相对时间（如“2小时前”）。
  3. 正文：展示文字内容。
  4. 图片九宫格：将正文中的 ![[xxx.png]] 转换为 Obsidian 可渲染的真实路径。
     
     样式要求：
     - 1张图：按原始比例缩放展示，设定最大宽高度。
     - 2至4张图：2x2 网格。
     - 5至9张图：3x3 网格。
     - 图片需使用 object-fit: cover 保证等比裁切。
  5. 底部交互区：包含“详情”按钮和“评论”按钮。
  6. 评论区展示：遍历元数据中的“评论”数组，在 Card 底部渲染评论列表。

5. 核心功能三：互动更新逻辑 (Interactivity)
- 跳转源文件：点击记录的“详情”按钮，调用 Obsidian API 跳转并打开对应的 .md 源文件。
- 添加评论：
  1. 点击“评论”按钮，在当前 Card 底部展开内联输入框。
  2. 用户输入文字并提交后，构造评论字符串（例如："2026-02-28 10:00:00 - 评论内容"）。
  3. 调用 app.fileManager.processFrontMatter() API，安全地将新评论追加到源文件的“评论”数组中，并更新“更新时间”。
  4. 数据更新完成后，局部重新渲染该 Card 的评论区 DOM。
